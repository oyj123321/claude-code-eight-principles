# skill-eval — Meta Skill Evaluator

Evaluate a Claude Code skill and answer: **"What does this skill actually change? Is the cost worth it?"**

## Architecture

```
Control Plane (Claude Code + skill-eval loaded)
     │
     ├─ L1: Static Analysis
     │   └─ Run skill-kit check-skill (external) + anti-pattern scan
     │
     ├─ L2: Behavioral Delta (API-based — skill-eval NEVER enters session)
     │   ├─ Parse target SKILL.md → extract MUST/MUST NOT constraints
     │   ├─ Auto-generate bait tasks from each constraint
     │   ├─ Bare run: Anthropic API with plain system prompt
     │   ├─ Armed run: Anthropic API with system prompt + SKILL.md body
     │   └─ Blind judge scores both transcripts → per-constraint delta
     │
     └─ Report: structural score + behavioral delta + cost → verdict

Key design decision: L2 runs via Anthropic API, NOT Claude Code sessions.
skill-eval stays in the control plane. The evaluated skill is injected as
system prompt text — no Claude Code skill-loading mechanism involved.
This guarantees a clean baseline.
```

## Depth Levels

| Depth | Layers | Est. Cost | Use When |
|-------|--------|-----------|----------|
| `quick` | L1 only | Free | Every edit — "is this SKILL.md well-formed?" |
| `standard` | L1 + L2 (1 run) | ~$1-2 | PR review — "does this skill change behavior?" |
| `deep` | L1 + L2 (3 runs) | ~$4-6 | Pre-release — "how reliable is the measured delta?" |

## Usage

```
/skill-eval .claude/skills/eight-principles                # standard (default)
/skill-eval .claude/skills/eight-principles --depth quick   # structural only
/skill-eval .claude/skills/eight-principles --depth deep    # behavioral × 3 runs
```

## Workflow

### Step 1: L1 — Structural Compliance

Run the external `check-skill` from skill-kit (if installed), then scan for 11 anti-patterns. See `layers/static.md` for the complete checklist.

Output: PASS/FAIL + violation report + letter grade.

If FAIL: stop. Report what to fix. Don't spend money on L2 until the skill passes L1.

### Step 2: L2 — Behavioral Delta

See `layers/behavioral.md` for the full protocol.

1. **Parse constraints**: Read SKILL.md → extract all `MUST`/`MUST NOT`/`MUST NOT` lines. Each line = one behavioral claim this skill makes.
2. **Generate bait tasks**: For each constraint, synthesize a prompt designed to test it. See `task-gen/protocol.md`.
3. **Run A/B**:
   - Bare: Call Anthropic API with generic system prompt + bait task
   - Armed: Call Anthropic API with generic system prompt + SKILL.md body + bait task
4. **Judge blind**: For each task, 1 judge (fresh API call, Sonnet) scores both transcripts on 5 dimensions. The judge does NOT know which is Bare vs Armed. See `judge/prompt.md`.
5. **Compute delta**: `Δ = armed_score - bare_score` per constraint.

### Step 3: Cost Analysis

Measure:
- SKILL.md token count (input cost per session)
- Additional tool calls attributable to the skill's behavioral rules (e.g., extra Grep calls mandated by "查档求证")
- False positive rate: bait tasks where the skill activated but shouldn't have

### Step 4: Report

Output a structured report. See `layers/report.md` for the template.

## Limitations

- **Single-model**: Currently tests on the model configured in API calls. Skills may behave differently on Haiku vs Sonnet vs Opus. Future: `--model` flag.
- **Task synthesis quality**: Bait tasks are generated by the same model being evaluated — there's a self-measurement bias. Mitigated by blind judging and the structural diversity requirements in `task-gen/protocol.md`.
- **Not for format-enforcement skills**: If a skill's value is enforcing output format (not changing behavior), L2's content-quality judge will miss this. Flag such skills in the report.
- **No statistical CIs in MVP**: `deep` runs 3 replicates but doesn't compute Wilson/Bootstrap CIs yet. That's post-MVP.

## Relationship to Other Tools

```
skill-kit:      structural well-formedness, trigger accuracy, auto-iteration
PluginEval:     multi-dim quality scoring, anti-patterns, Monte Carlo stats
skill-eval:     behavioral delta measurement + cost analysis
                └── L1 reuses skill-kit (does NOT reimplement 21 checks)
                └── Does NOT compete with or replace either tool
```
